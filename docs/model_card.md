# Model Card

## Model Name
Falabella Hybrid Risk Ensemble (XGBoost over tabular + GraphSAGE embeddings)

## Version
v1.0 (2026-04-13)

## Intended Use
Decision support for thin-file applicant risk triage in micro-credit scenarios.
The model is intended for loan officer assistance, not fully autonomous final lending decisions.

## Training Data
- Source: synthetic borrower, repayment, graph, and behavioral signal tables generated by `src/falabella_risk/data/generate_data.py`
- Feature table: `data/processed/features.parquet`
- Graph embeddings: `data/processed/gnn_embeddings.parquet`
- Label: `default_flag`

## Performance (Hold-out Test)
- AUC: 0.9817
- PR-AUC: 0.9263
- F1: 0.8504
- Brier score: 0.0422
- Inference latency: 0.0078 ms per row

## Fairness Evaluation

### Sensitive Slices

- `indigenous_proxy`

### Mitigation Applied
- Strategy: intersectional threshold calibration
- Outcome (after mitigation):
	- gender parity gap: 0.0016

	- indigenous proxy parity gap: 0.0450
- Threshold target achieved: all parity gaps < 0.05

## Explainability
- Local explanation API: `src/falabella_risk/evaluation/explainability.py` (`predict_explain`)
- Output payload includes:
	- risk score
	- decision threshold
	- top drivers with contribution direction and magnitude
- Streamlit dashboard includes top-driver table and SHAP-style waterfall chart.

## Cold-Start Routing
- Phase logic implemented in `src/falabella_risk/inference/cold_start.py`:
	- Month 1: rule-based scorer
	- Months 2-3: hybrid-lite fallback
	- Month 3+: full hybrid model

## Federated Simulation
- Implemented in `src/falabella_risk/federated/federated_training.py`
- FedAvg clients: CDMX, Monterrey, Guadalajara, Yucatan
- Federated AUC: 0.9030
- Centralized reference AUC: 0.8984
- AUC gap: -0.0046 (within 2-point target)
- Report artifact: `reports/federated_report.json`

## Governance and Compliance Notes
- Designed to support explanation and audit trace requirements under LFPDPPP 2025 expectations for automated decision transparency.
- Intended for operational alignment with CONDUSEF-oriented consumer protection and review workflows.
- Dashboard includes override logging to support human-in-the-loop governance.

## Limitations
- Synthetic-only corpus limits external validity and may not reflect production drift.
- Fairness calibration effectiveness can change under real-world class imbalance shifts.
- Economic stress regimes and policy shocks are not explicitly modeled.

## Recommended Monitoring
- Monthly subgroup parity and calibration checks.
- Population stability and feature drift monitoring.
- Override log review for systematic disagreement patterns.
